# Афиша Costa Blanca — Критический анализ проекта

## 1. Оценка идеи

### Сильные стороны
- **Чёткая ниша**: русскоговорящие экспаты на Costa Blanca (50K-100K+ человек, сконцентрированы в Торревьехе, Бенидорме, Аликанте, Кальпе, Альтее)
- **Реальная боль**: события публикуются только на испанском, разбросаны по десяткам муниципальных сайтов, теряются в чатах Telegram
- **Нулевая конкуренция**: НЕТ ни одного структурированного агрегатора событий на русском для Costa Blanca
- **Низкий порог входа**: не нужен контент с нуля — агрегация + перевод + курирование
- **Telegram = идеальный канал**: русскоязычная аудитория в Испании живёт в Telegram

### Слабые стороны / Риски
- **Зависимость от источников**: если сайты-источники поменяют структуру — парсеры ломаются
- **Сезонность**: летом событий много, зимой — мало (но Коста Бланка — круглогодичный регион, риск умеренный)
- **Монетизация неочевидна**: на старте это бесплатный сервис, монетизация через рекламу/партнёрства позже
- **Одиночный проект**: зависит от одного человека (автоматизация критически важна)

### Критика текущего документа

| Проблема | Что не так | Рекомендация |
|----------|-----------|--------------|
| **n8n как ядро** | n8n — ещё одна система для поддержки, отдельный хостинг, отдельные расходы. Для парсинга + перевода это overkill | Заменить на Claude Code скрипты (Python/TS) + cron. Проще, дешевле, мощнее |
| **ChatGPT для обработки** | Зачем ChatGPT, если есть Claude API? Дороже, менее интегрировано | Использовать Claude API для перевода + форматирования |
| **Нет приоритизации источников** | Просто "costablancaup + fiestas.net" — слишком мало | 7 источников с приоритетами (см. ниже) |
| **Нет стратегии контента** | Парсинг + публикация, но нет tone of voice, формата постов | Нужен чёткий формат Telegram-поста с шаблоном |
| **SEO преждевременно** | meta_title/description для MVP — лишняя сложность | Убрать из MVP, добавить в v2 |
| **Слишком много полей в БД** | 40+ полей в таблице events — перегруз для MVP | Сократить до ~20 essential полей |
| **Нет аналитики** | Как понять, что работает? Какие категории популярны? | Добавить простой трекинг кликов |

## 2. Конкурентный ландшафт

### Прямые конкуренты (русский язык, Costa Blanca, события) = **НЕТ**

Ни одного структурированного сервиса. Существуют только:
- **Telegram-чаты** ("Русские в Аликанте", "Торревьеха русские") — события тонут в общем потоке
- **VK-группы** — теряют аудиторию, переходят в Telegram
- **spain.ru, espanarusa.com** — общие порталы об Испании, без фокуса на события Costa Blanca

### Косвенные конкуренты

| Конкурент | Что делает | Слабость |
|-----------|-----------|----------|
| Eventbrite Spain | Платные/билетные мероприятия | Нет русского, нет местных фестивалей |
| Fever.co | Кураторский гид | Только Мадрид/Барселона |
| costablancaup.com | Англоязычный блог о событиях | Нет русского, блог-формат без фильтров |
| hoyalicante.app | Ежедневная афиша провинции | Только испанский |
| Муниципальные сайты | Официальные события | Каждый город отдельно, только испанский |

### Вывод: **голубой океан**
Проект занимает незанятую нишу. Ближайший конкурент — costablancaup.com (англоязычный). Русскоязычного аналога нет.

## 3. Источники данных (приоритизированные)

### Tier 1 — Старт (первая неделя)

| # | Источник | Доступ | Что даёт |
|---|---------|--------|----------|
| 1 | **alicante.es/es/agenda** | RSS-фид `/agenda.xml` | Официальные события Аликанте, 25 позиций, структурированный XML |
| 2 | **costablancaup.com** | RSS `/feed/`, WordPress | Фестивали, рынки, культура всей Costa Blanca. EN/ES |
| 3 | **comunitatvalenciana.com** | REST API с фильтрами | Туристические события по провинции, мультиязычный |

### Tier 2 — Вторая неделя

| # | Источник | Доступ | Что даёт |
|---|---------|--------|----------|
| 4 | **hoyalicante.app** | JSON-LD, скрапинг | Ежедневные обновления, вся провинция |
| 5 | **Eventbrite API** | Публичный API, OAuth | Билетные мероприятия: концерты, воркшопы |
| 6 | **agendacultural.diputacionalicante.es** | WordPress, скрапинг | Культурные события 141+ муниципалитетов |

### Tier 3 — Расширение

| # | Источник | Доступ | Что даёт |
|---|---------|--------|----------|
| 7 | **alicanteout.com** | RSS, WordPress | Музыка, театр, кино, дети, еда |
| 8 | **Meetup.com** | GraphQL API | Социальные события, экспат-тусовки |

### Источники, которые НЕ стоит использовать
- **informacion.es** — блокирует ботов, paywall
- **fiestas.net** — crawl-delay 50 сек, ненадёжные URL
- **costablanca.org** — сломанный Angular-сайт
- **Facebook** — API закрыт для скрапинга событий с 2018

## 4. Архитектура без n8n (чисто Claude Code)

```
┌─────────────────────────────────────────────────────┐
│                   АВТОМАТИЗАЦИЯ                      │
│              (Claude Code + Python/TS)               │
│                                                     │
│  cron-скрипты (запуск по расписанию):               │
│                                                     │
│  1. fetch-events.py     — парсинг 7 источников      │
│  2. process-events.py   — Claude API: перевод,       │
│                           форматирование, оценка     │
│  3. publish-telegram.py — публикация в Telegram      │
│  4. archive-events.py   — архивация прошедших        │
│                                                     │
│  Запуск: GitHub Actions / Vercel Cron / локальный   │
│          cron на VPS                                │
└──────────────┬──────────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────────┐
│                    SUPABASE                          │
│                                                     │
│  - PostgreSQL (события, категории)                  │
│  - Auth (админ)                                     │
│  - Storage (фото)                                   │
│  - Realtime (обновления на сайте)                   │
│  - Edge Functions (webhook для Telegram)            │
└──────────────┬──────────────────────────────────────┘
               │
        ┌──────┴──────┐
        ▼             ▼
┌──────────────┐ ┌──────────────┐
│  Сайт        │ │  Telegram    │
│  (Lovable)   │ │  канал       │
│              │ │              │
│  React +     │ │  Бот         │
│  Supabase    │ │  публикует   │
│  client      │ │  посты       │
└──────────────┘ └──────────────┘
```

### Преимущества перед n8n:
- **0 дополнительных расходов** на хостинг n8n ($20-50/мес экономия)
- **Полный контроль** — код в репозитории, версионирование, тесты
- **Claude API** вместо ChatGPT — один провайдер, одна экосистема
- **GitHub Actions** для cron — бесплатно до 2000 мин/мес
- **Проще дебажить** — логи, ретраи, error handling в коде

## 6. Рекомендуемый MVP (4 недели)

### Неделя 1: Фундамент
- Supabase: БД + Auth + Storage
- Lovable: сайт (главная + каталог + страница события + админка)
- Telegram: канал + бот

### Неделя 2: Автоматизация
- Python-скрипты парсинга (Tier 1 источники)
- Claude API интеграция (перевод + форматирование)
- Telegram Bot API (публикация)

### Неделя 3: Контент + тестирование
- Наполнение: 20-30 событий вручную + автоматически
- Тестирование полного цикла
- Доработка дизайна

### Неделя 4: Запуск
- Подключение домена
- Публикация в русскоязычных группах
- Первый дайджест "Что делать на выходных"
- Подключение Tier 2 источников
