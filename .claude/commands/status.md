# /status

Показывает статус всех постов Костоплана. Читает все `content/schedule/week*.json` и выводит таблицу.

## Использование

- `/status` — все недели
- `/status w5` — только week5
- `/status today` — посты на сегодня
- `/status upcoming` — approved и неопубликованные в пределах 7 дней
- `/status missing` — что ещё в draft и близко по дате (alarm)

## Формат вывода

```
week │ id         │ date       │ time  │ type             │ status    │ approved
─────┼────────────┼────────────┼───────┼──────────────────┼───────────┼─────────
4    │ w4-wed-1   │ 2026-04-22 │ 10:00 │ weekly_digest    │ draft     │ no
4    │ w4-wed-2   │ 2026-04-22 │ 18:00 │ markets_week     │ draft     │ no
4    │ w4-thu-1   │ 2026-04-23 │ 10:00 │ event_preview    │ draft     │ no
...
5    │ w5-fri-1   │ 2026-05-01 │ 10:00 │ today_holiday    │ approved  │ yes
─────┴────────────┴────────────┴───────┴──────────────────┴───────────┴─────────

Итого по status:
  draft:     28
  approved:   6
  published: 18
  failed:     0

Требует внимания:
  ⚠️ 3 поста в draft с датой в пределах 7 дней:
     w4-wed-1 (2026-04-22 10:00) — завтра!
     w4-wed-2 (2026-04-22 18:00) — завтра!
     w4-thu-1 (2026-04-23 10:00) — через 2 дня
```

## Когда запускать

- Утром — что публикуется сегодня
- Перед `/approve` — что уже в статусе approved
- После автопубликации — что ушло, что зафейлилось
- Раз в неделю — чтобы видеть, где провалы в плане
