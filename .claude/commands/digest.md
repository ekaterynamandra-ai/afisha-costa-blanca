# /digest

Создаёт большой месячный дайджест — обзор всех событий месяца. По правилам из `docs/editorial-calendar.md` выходит **в последнюю пятницу предыдущего месяца**.

## Использование

- `/digest may` или `/digest 2026-05` — дайджест мая
- `/digest june` — июнь
- `/digest next` — следующий месяц (от сегодня)

## Что делает

1. Читает события из:
   - `content/events/events-april-may-2026.json`
   - `content/data/<month>-2026-events.md`
   - `content/google-maps-export/` (для мест)
2. Применяет контент-политику: отсеивает испанские муз/кино фестивали, длинные походы, слабые платные события
3. Группирует оставшееся:
   - 🎭 Крупные фестивали / праздники
   - 🎉 Локальные фиесты по городам
   - 🛍 Рынки (когда, где)
   - 🌸 Сезонное (что цветёт, какие фрукты, фламинго)
   - 🏖 Пляжи / природа (новости сезона)
   - 📅 Национальные выходные
4. Формирует пост: заголовок + 5-7 блоков + CTA `📱 @afishaCB — каждую неделю подборки!`
5. **Добавляет в расписание** на последнюю пятницу предыдущего месяца, 19:00
6. Ставит `status: draft`, ждёт одобрения

## Тайминг публикации

| Дайджест месяца | Публикация |
|-----------------|------------|
| Апрель | пт 28 марта 19:00 |
| Май | пт 24 апреля 19:00 |
| Июнь | пт 29 мая 19:00 |
| Июль | пт 26 июня 19:00 |

## Длина

- Цель: 1000-1500 символов (укладывается в 1 сообщение Telegram с фото)
- Если больше — разбить на 2 поста (часть 1/2) или убрать второстепенное
- Каждое событие — 2-3 строки максимум
