# /test_mode

Тестирует автопубликацию через систему: отправляет пост админу в личку (НЕ в канал @afishaCB).

## Что делает

1. Если передан ID существующего поста — добавляет ему `test_mode: true`, оставляет статус `approved`
2. Если ID не передан — создаёт быстрый тестовый пост в расписании на ближайшие 5 минут
3. Запускает GitHub Actions вручную или ждёт следующего cron
4. Пост приходит **тебе в личку** с префиксом "🧪 ТЕСТ:"
5. После теста — убирает `test_mode` или удаляет тестовый пост

## Использование

- `/test_mode` — создать новый тестовый пост
- `/test_mode 13` — протестировать существующий пост по ID
- `/test_mode publish` — протестировать публикацию в реальном времени (запустить Actions)
- `/test_mode notify` — протестировать только систему уведомлений
- `/test_mode reminder` — протестировать напоминания о планировании

## Что проверять при тесте

- Пришёл ли пост в личку (не в канал!)
- Корректно ли отображается фото/альбом
- Работают ли HTML-теги (b, a, i)
- Пришёл ли отчёт от бота
- Сохранился ли статус в JSON

## После теста

- Убрать `test_mode: true` из расписания
- Если был временный cron в workflow — убрать
- Удалить тестовый пост из истории
