## Доступные команды

- **`/test_mode`** — тестировать автопубликацию через личку админа
- **`/preview`** — создать HTML-превью постов на неделю (в content/drafts/)
- **`/approve <week>`** — одобрить план на неделю (поставить `approved: true`)
- **`/publish_now <id>`** — опубликовать конкретный пост вручную сейчас
- **`/status`** — показать статус всех постов в расписании
- **`/digest <month>`** — создать месячный дайджест по неделям

## Правила работы

### Контент-политика
- НЕ публиковать без одобрения пользователя
- НЕ публиковать испаноязычные музыкальные/кино фестивали (низкий интерес)
- НЕ публиковать длинные походы 8+ км
- ДА: гастро-фестивали, парады, бесплатные семейные события, природа, рынки
- Платные события — только самые яркие
- Все локации — Google Maps ссылки на конкретные места (не координаты)
- Каждый пост — с фото или альбомом (не Wikipedia, лучше Pexels или официальные сайты)

### Тайминг публикаций
- Месячный дайджест: последняя пятница месяца
- Недельный дайджест: суббота
- Подборка выходных: среда вечер
- Акценты: за 3-7 дней до события
- Время в расписании = испанское (Europe/Madrid)

### Перед публикацией всегда
1. Показать превью HTML
2. Дождаться одобрения пользователя
3. Поставить `approved: true` в week*.json
4. GitHub Actions опубликует автоматически

## Скрипты

- **autopublish.py** — публикует посты по расписанию (status=approved + время наступило). Запускается через GitHub Actions раз в 2 часа.
- **publish_telegram.py** — ручная публикация одного поста.

## Cron расписание (UTC → CEST испанское)
- 06:00 UTC = 08:00 CEST (рынок утро)
- 08:00 UTC = 10:00 CEST (утренние посты)
- 12:00 UTC = 14:00 CEST (дневные)
- 16:00 UTC = 18:00 CEST (вечерние)
